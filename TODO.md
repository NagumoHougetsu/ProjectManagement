# WBSツール 開発TODO・経緯まとめ

## 過去の経緯と基盤移行の歴史
1. **初期 (Vis.js ベース)**
   - 柔軟な配置（同じレーンに複数のタスクを自由に置ける、自由な上下移動）などが強みだった。
   - しかし、DOM構造がブラックボックス化されており、マウスホバー時の行列ハイライトなどの細かい要件を追加する際にエラー（循環参照など）が多発・クラッシュする問題が発生した。

2. **DHTMLX Gantt への移行試行 (app_v2.js)**
   - より安定した高機能ライブラリとしてDHTMLXへの移行を試みた。
   - しかし、DHTMLXは「表計算ソフト的」な厳密な木構造を前提としており、「同じ行に複数のタスクを自由に配置して上下にドラッグ移動する」というVis.js時代の柔軟な操作感を再現できない（仕様上不可能に近い）ことが判明した。

3. **完全自作基盤への移行 (app_v3.js / 現在)**
   - ブラックボックス化を避け、要件を100%コントロールするために「Vanilla JS + Tailwind CSS」でガントチャート自体を完全自作する方針へ切り替えた。
   - 1日もかからず基本機能は実装できたが、ライブラリが担っていた細かい使い勝手（ズーム、スナップ、イベント判定など）が失われたため、細かな「先祖返り」が発生し、それを手動で逐一直していく作業が発生している。
   - さらに「Pythonサーバーがプロセスに居座り続け、変更が反映されない」というトラブルがあり、余計な工数とAPIコストを消費してしまった。

## 追加実装予定の機能 (New Features)
- [x] **マイルストーン（セクションごとの〆切）の表示・操作機能** [優先度: 1]
  - キャラクター（子グループ）の範囲内で、各セクション（モデル、モーション、エフェクト等）の〆切日を示す縦線（セクションカラーの太線）を表示する。
  - 線の上部に「▼」マークを配置し、それをドラッグして左右にスライドさせることで〆切日をインタラクティブに変更できる仕様にする。
- [x] **バージョン（親グループ）の期間連動と制約（コンストレイント）** [優先度: 2]
  - バージョンのバーは、子タスクの開発期間に合わせて自動で伸縮・追従するようにする。
  - ただし、終了日はマスターデータにある「アート締め」の日付でコンストレイント（固定・上限制約）されるようにする。
- [x] **フィルタリング機能・検索機能の拡充** [優先度: 3]
  - 担当者ごとのフィルタリング、テキスト検索、完了（100%）済みタスクの非表示機能など。
  - **【完了済みの機能詳細】**
    - 自由検索欄: タスク名、キャラ名、バージョン名、担当者名など全項目を対象に部分一致検索。
    - 担当者フィルター: カスタムのドロップダウンUIを実装し、名前による絞り込みと複数選択が可能。
    - 挙動変更: 条件に合致しないタスクは非表示とし、空いたスペースに表示対象タスクが上に詰まって（テトリス配置されて）表示されるように実装。
    - エディタ改善: 必須条件（バージョン、キャラ名、セクション等）を満たさない場合、「適用」ボタンをグレーアウトして不正な空タスクの生成を防止。
- [x] **複数タスクの一括操作** [優先度: 4]
  - 複数選択（Ctrl+クリックなど）によるタスクの同時移動機能。
- [x] **グループ（親/子）の上下ドラッグ移動機能** [優先度: 4.5]
  - 子グループ（キャラクター）自体を掴んで、上下にドラッグして順序を入れ替える機能。（親グループの移動は不要）

  - 左右（日付）方向への移動はタスクに依存するため不可とし、上下移動のみとする。
  - 開閉（折りたたみ/展開）のトグルは「左端の▼アイコン周辺」をクリックした時のみに限定する。
  - 移動した順序は即時保存ではなく、「保存」ボタンを押した時にまとめて更新（タスク順/マスター）する。
