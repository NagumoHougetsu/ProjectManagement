# WBSツール 開発TODO・経緯まとめ

## 過去の経緯と基盤移行の歴史
1. **初期 (Vis.js ベース)**
   - 柔軟な配置（同じレーンに複数のタスクを自由に置ける、自由な上下移動）などが強みだった。
   - しかし、DOM構造がブラックボックス化されており、マウスホバー時の行列ハイライトなどの細かい要件を追加する際にエラー（循環参照など）が多発・クラッシュする問題が発生した。

2. **DHTMLX Gantt への移行試行 (app_v2.js)**
   - より安定した高機能ライブラリとしてDHTMLXへの移行を試みた。
   - しかし、DHTMLXは「表計算ソフト的」な厳密な木構造を前提としており、「同じ行に複数のタスクを自由に配置して上下にドラッグ移動する」というVis.js時代の柔軟な操作感を再現できない（仕様上不可能に近い）ことが判明した。

3. **完全自作基盤への移行 (app_v3.js / 現在)**
   - ブラックボックス化を避け、要件を100%コントロールするために「Vanilla JS + Tailwind CSS」でガントチャート自体を完全自作する方針へ切り替えた。
   - 1日もかからず基本機能は実装できたが、ライブラリが担っていた細かい使い勝手（ズーム、スナップ、イベント判定など）が失われたため、細かな「先祖返り」が発生し、それを手動で逐一直していく作業が発生している。
   - さらに「Pythonサーバーがプロセスに居座り続け、変更が反映されない」というトラブルがあり、余計な工数とAPIコストを消費してしまった。

## 追加の改善事項（完了）
- **右クリックコンテキストメニューの配置バグ修正**: スクロールや相対配置の影響でメニューがマウスポインタのずっと下に表示されてしまう問題を、`position: fixed` と `clientX/clientY` の採用によって解決（常にマウスポインタの真横・最前面に表示）。

  - **【完了済みの機能詳細】**
    - 表示ON/OFFのトグルボタンを追加
    - 遅延（左折れ）は赤、順調（右折れ）は青で色分け
    - 本日からの日数差に応じて線の濃さ（不透明度）を動的に変更
    - 差分が7日未満の場合は破線、7日以上の場合は実線で深刻度を視覚化
- [x] **先行後続タスク（依存関係）のUI実装** [優先度: 8]
  - タスク同士を線で繋ぐ機能。
  - **【完了済みの機能詳細】**
    - タスクの右端にホバーすると、赤い丸の「＋」コネクタが出現。
    - コネクタからドラッグすることで、リアルタイムにベジェ曲線が追従する「プレビュー線」をSVGで美しく自動描画。
    - 別のタスクの左端またはタスクそのものにマウスが近づくと対象タスクが緑色にハイライト（スナッププレビュー）され、ドロップすることで直感的に依存関係（結線）を登録。
    - 循環依存（矛盾する無限ループ構造）を検知した場合には、警告アラートを出して結線を防止。
    - 結線された依存関係線を右クリックすると、専用コンテキストメニューから「この依存関係を削除」できるUIを完備。
    - 保存ボタンで他のスケジュール同様にバックエンド（CSV）に `dependencies` カラムとしてシームレスに保存・永続化。
    - **【追加の高度な制約・連動機能】**:
      - **押し出し（連動シフト）**: 先行タスクを右（未来）へ移動して後続タスクの開始日を超えた場合、超えた日数分だけ後続タスク（およびその先の後続タスクも再帰的）を自動的に右へ押し出すドミノシフト機能を実装。
      - **左移動制限**: 後続タスクを過去（左）に動かそうとする際、先行タスクの終了日より前には絶対に配置できないように、ドラッグ/リサイズ中の座標を自動クランプ制御。
      - **複数矩形選択（範囲選択）**: ShiftキーまたはCtrlキーを押しながらガントチャートの空白をドラッグすることで、半透明なブルー of 点線選択ボックスを表示し、囲んだタスクを一瞬で一括選択できる矩形（ドラッグ）選択機能を実装。島（タスク群）のまとめての移動が劇的にスムーズに行えるようになりました。
