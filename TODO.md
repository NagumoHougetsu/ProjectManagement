## 🚀 次のステップ (NEXT TODO)

次のセッションで対応・調整すべき課題です。

### 1. 日付列幅の極限縮小対応（最優先）
- **現状**: 日付の列幅がまだ広く、縮小（ズームアウト）していくとすぐに1W/2Wなどの週表記に切り替わってしまうか、文字が間引かれてしまう。
- **目標**: 日付列の幅を現在の1/5程度まで細く維持し、ギリギリまで「1日単位（1, 2, 3...）」の表示を粘らせる。
- **実装ヒント**: 
  - `style.css` の `.vis-time-axis .vis-text` に対して、`transform: scaleX(0.8);` などのさらなる縮小を試すか、`min-width` をさらに小さくする。
  - `app.js` の `zoomMin` の値をさらに小さく調整し、限界までズームアウトできるようにする。
  - Vis.jsの `timeAxis: { scale: 'day', step: 1 }` を強制しつつ、ズームイベント（`rangechange`）をフックして、ある閾値で動的に `options` を書き換えて週表記にスイッチする高度な実装を検討する。

### 2. ガントバーの密接具合・重なり修正
- **現状**: CTRL+ホイールで高さを変更した際や、ズームレベルによってはバーの余白にばらつきが出る。
- **目標**: バー同士をもっと密接させたい。
- **実装ヒント**: `app.js` の `margin: { item: { vertical: ... } }` の値を調整する。また、高さ変更イベント時に `timeline.setOptions()` だけでなく、完全にDOMを再計算させるアプローチを探る。

### 3. CSVとGit管理の最適化
- サンプル用のマスターデータ（`data/projects/Sample/*`）はGitにコミットするため、`.gitignore` の設定を見直し、特定のプロジェクト（Sample）だけはGit追跡対象にするなどの工夫が必要。
